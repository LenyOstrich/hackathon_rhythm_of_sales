# Решение хакатона "Ритм продаж"

## Описание задачи
Задача — спрогнозировать ежедневные продажи товаров (кроссовок) на маркетплейсе, учитывая цену, остатки на складе и участие в промо. Нужно предсказать количество проданных единиц для каждой пары товар-день в тестовом периоде.

## Данные
- **train.parquet** — история продаж: `nm_id`, `dt`, `price`, `is_promo`, `prev_leftovers`, `qty` (целевой признак)  
- **test.parquet** — 14 дней после train, без `qty`  
- **sample_submission.csv** — шаблон для отправки решения

## Метрика
Взвешенная средняя абсолютная ошибка (wMAE):  

wMAE = (Σ wᵢ · |yᵢ - ŷᵢ|) / Σ wᵢ  

где  
- wᵢ = 1, если yᵢ = 0  
- wᵢ = 7, если yᵢ > 0  

Вес 7 для дней с продажами подобран так, чтобы суммарный вес классов был примерно равным (доля дней с продажами ≈ 0.13).

## Подход
Двухэтапная модель на CatBoost:

1. **Классификатор** — предсказывает факт наличия продаж (sale > 0)  
   - iterations=2000, depth=6, lr=0.04  
   - class_weights=[1, 7], loss=Logloss, early_stopping=120  

2. **Регрессор** — предсказывает log₁ₚ(qty) только для дней, где продажи > 0  
   - iterations=1200, depth=6, lr=0.05  
   - loss=MAE  

## Результат
- wMAE на private leaderboard: **0.339**
